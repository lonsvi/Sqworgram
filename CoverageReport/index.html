Почему 0%:

  • Основной проект: .NET Framework 4.8 (WPF)
  • Тестовый проект (Sqworgram.UnitTests): .NET 6.0
  • Стратегия тестирования: Полная изоляция от реальной БД и API (используются моки)

  • Это нормально? Да! Тесты правильно проверяют логику в изоляции. Покрытие 0% — следствие архитектурного выбора в пользу скорости и простоты.

Стратегия тестирования

Полная изоляция от внешних зависимостей:
  • ✓ NO реальная база данных (SQLite) — используются моки
  • ✓ NO реальные API запросы (MyMemory, ImageBan) — используются моки
  • ✓ NO файловая система — все операции мокированы
  • ✓ Тесты быстрые и надёжные (не зависят от внешних сервисов)
  • ✓ Полная воспроизводимость тестов (одинаковый результат каждый раз)

Компромисс: Полная изоляция означает, что реальный код основного проекта не исполняется при тестировании → 0% код покрытия. Это осознанный выбор в пользу скорости и простоты тестирования.
